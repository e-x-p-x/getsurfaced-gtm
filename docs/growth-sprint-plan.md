## Kill / Scale Decision Framework (Apply After Sprint 2)

### Leading Indicators by Hack Type

| Hack Type | Leading Indicator | Minimum Threshold (2 weeks) |
|-----------|------------------|-----------------------------|
| Outbound (email/LinkedIn) | Reply rate | > 5% |
| Content (LinkedIn, blog) | Engagement rate | > 2% or 500+ impressions/post |
| Free tool / audit | Visits → email capture | > 10% conversion |
| Partnerships / referrals | Partners activated | > 2 active |
| Community / seeding | Inbound leads | > 3 from channel |
| Paid / retargeting | Cost per lead | < €30 CPL |
| Product-led (demo, widget) | Activation rate | > 5% of visitors |

### Verdict Rules

- Leading indicator >= 2x threshold → **SCALE** (double budget/time)
- Leading indicator >= threshold → **ITERATE** (tweak messaging/targeting)
- Leading indicator 50-99% threshold → give 1 more week
- Leading indicator < 50% threshold → **KILL** (reallocate hours immediately)

### Reallocation Priority

1. Scale winners (more volume, more budget)
2. Launch next hack from backlog
3. Never spread thin across everything
